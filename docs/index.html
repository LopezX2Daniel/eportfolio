Unit 11 project versus Unit 6

There are significant differences between the Unit 6 project and the Unit 11 project. Although both involve Machine Learning exercises, there are two clear distinctions.

First, the Unit 6 project was a team effort, which helped distribute the workload and provided more hands to work toward the same goal, in contrast, Unit 11 was an individual assignment where full responsibility and progress rested solely on me.

Second, Unit 6 involved a classic Machine Learning case study, specifically regression and clustering, which reduced the complexity. For Unit 11, however, I had to implement a convolutional neural network model, a concept entirely new to me. This required me to first research, read, and learn about this type of Deep Learning before writing the network's code, ensuring I understood the function of each part of the code and how hyperparameters influenced the final result.

From a technical standpoint, there is a major difference between analyzing time series data and images (which are ultimately just pixels). The choice of model depends on the type of data being analysed. For instance, when training a Random Forest model in classic ML, the "RandomForestRegressor" library is used for time series prediction, whereas "RandomForestClassifier" is used when predicting Boolean output values, that is, "True" or "False."

Similarly, model configuration and hyperparameter tuning tend to be far more complex in Deep Learning than in classic ML. There is no strict formula for convolutional network architecture that guarantees satisfactory results every time. Decisions regarding the number of convolutional or pooling layers, pixel dimensions, or whether to include dropout layers depend entirely on the input data and how the model behaves with each modification. Not to mention that available hardware is another variable that must be taken into account.

Debugging also proved to be a different experience between both projects. In Unit 6, when an issue is detected, it was often easier to identify since classic ML models are more interpretable and their outputs more predictable. In Unit 11, troubleshooting a convolutional neural network was considerably more challenging, as errors could stem from multiple sources, whether from the data itself, the architecture, or the hyperparameters, making it harder to pinpoint the exact cause.

Beyond the technical skills, Unit 11 pushed me to develop greater autonomy and problem-solving abilities, since I could not rely on teammates to fill knowledge gaps or catch mistakes. This experience strengthened my confidence in tackling unfamiliar topics independently, a skill I consider valuable for future projects.

I really enjoyed working on both projects, the opportunity to work in a team and meet new classmates during Unit 6 was rewarding, and across both projects, I was able to go beyond ML theory and work with real-world examples. In fact, this motivated me to apply ML models in my work, given that there are currently no initiatives in this area.
